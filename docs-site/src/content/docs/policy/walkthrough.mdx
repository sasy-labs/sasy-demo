## Three Representations

<Tabs>
  <TabItem label="English">
    ```markdown
    # Airline Booking Policy

    ## Default behavior
    - All actions are authorized by default

    ## Cancellation policy
    - Gold members can cancel any reservation,
      regardless of insurance status
    - Silver and regular members can only cancel
      if the reservation has travel insurance
    - Deny cancellation if the reservation has
      no insurance and the member is not gold

    ## Flight modification policy
    - Economy reservations can be modified by any member
    - Basic economy reservations can only be modified
      by silver or gold members
    - Deny flight modifications on basic economy
      reservations for regular members
    ```
  </TabItem>
  <TabItem label="Structured Spec">
    ```
    C1: Authorize all actions by default
    C2: Allow cancel_reservation if membership = gold
    C3: Allow cancel_reservation if insurance = yes
    C4: Deny cancel_reservation if insurance = no
        AND membership != gold
    C5: Guard — deny cancel_reservation if
        reservation details not looked up
    C6: Allow update_reservation_flights if
        cabin = economy
    C7: Allow update_reservation_flights if
        cabin = basic_economy AND membership
        in (silver, gold)
    C8: Deny update_reservation_flights if
        cabin = basic_economy AND membership = regular
    C9: Guard — deny update_reservation_flights if
        reservation details not looked up
    ```
  </TabItem>

## Clause-by-Clause Breakdown

Each clause maps English intent to a concrete
Datalog rule. Click to expand.

---

<details>
<summary>**C1: Authorize all actions by default**</summary>

**English:** "All actions are authorized by default"

**Datalog:**
```prolog
IsAuthorized(idx) :- Actions(idx, _).
```

Every tool call is allowed unless a specific
`Unauthorized` rule overrides it. Without this
rule, tools like `get_reservation_details` would
be blocked — even though the policy has no rules
about them.

</details>

---

### Cancellation Rules

<details>
<summary>**C2: Allow cancel if gold member**</summary>

**English:** "Gold members can cancel any
reservation, regardless of insurance status"

**Datalog:**
```prolog
IsAuthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "cancel_reservation"),
    ToolResultField("get_reservation_details",
        "membership", "gold").
```

Checks the `membership` field from
`get_reservation_details`. Gold overrides the
insurance check entirely.

</details>

<details>
<summary>**C3: Allow cancel if has insurance**</summary>

**English:** "Silver and regular members can only
cancel if the reservation has travel insurance"

**Datalog:**
```prolog
IsAuthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "cancel_reservation"),
    ToolResultField("get_reservation_details",
        "insurance", "yes").
```

Any member with insurance can cancel, regardless
of tier.

</details>

<details>
<summary>**C4: Deny cancel if no insurance, not gold**</summary>

**English:** "Deny cancellation if the reservation
has no insurance and the member is not gold"

**Datalog:**
```prolog
// @deny_message: Cannot cancel without insurance
//   unless you are a gold member
// @suggestion: Consider adding travel insurance
// @tool_pattern: cancel_reservation
Unauthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "cancel_reservation"),
    ToolResultField("get_reservation_details",
        "insurance", "no"),
    !ToolResultField("get_reservation_details",
        "membership", "gold").
```

The `@deny_message` is shown to the agent when the
action is denied. The agent relays this to the
customer.

</details>

<details>
<summary>**C5: Guard — deny cancel if not looked up**</summary>

**Why:** The agent must call
`get_reservation_details` before attempting to
cancel. Otherwise it could bypass the insurance
check.

**Datalog:**
```prolog
// @deny_message: Look up reservation details first
// @tool_pattern: cancel_reservation
Unauthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "cancel_reservation"),
    !ToolResultField("get_reservation_details",
        "insurance", _).
```

The `!ToolResultField(..., _)` checks that the
lookup hasn't happened yet. This is a
**prerequisite guard** — a common pattern for
policies that depend on runtime data.

</details>

### Modification Rules

<details>
<summary>**C6: Allow modify if economy cabin**</summary>

**English:** "Economy reservations can be modified
by any member"

**Datalog:**
```prolog
IsAuthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "update_reservation_flights"),
    ToolResultField("get_reservation_details",
        "cabin", "economy").
```

</details>

<details>
<summary>**C7: Allow modify basic economy for silver/gold**</summary>

**English:** "Basic economy reservations can only
be modified by silver or gold members"

**Datalog:**
```prolog
IsAuthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "update_reservation_flights"),
    ToolResultField("get_reservation_details",
        "cabin", "basic_economy"),
    ToolResultField("get_reservation_details",
        "membership", "silver").

/* (similar rule for gold) */
```

</details>

<details>
<summary>**C8: Deny modify basic economy for regular**</summary>

**English:** "Deny flight modifications on basic
economy reservations for regular members"

**Datalog:**
```prolog
// @deny_message: Regular members cannot modify
//   basic economy reservations
// @suggestion: Upgrade membership or cabin class
// @tool_pattern: update_reservation_flights
Unauthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "update_reservation_flights"),
    ToolResultField("get_reservation_details",
        "cabin", "basic_economy"),
    ToolResultField("get_reservation_details",
        "membership", "regular").
```

</details>

<details>
<summary>**C9: Guard — deny modify if not looked up**</summary>

**Datalog:**
```prolog
// @deny_message: Look up reservation details first
// @tool_pattern: update_reservation_flights
Unauthorized(idx) :-
    Actions(idx, a),
    IsTool(a, "update_reservation_flights"),
    !ToolResultField("get_reservation_details",
        "cabin", _).
```

Same pattern as C5 — ensures the agent looks up
reservation details before attempting to modify.

</details>
