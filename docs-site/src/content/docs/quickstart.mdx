## Setup

<Steps>

1. **Clone and install**

   ```bash
   git clone https://github.com/sasy-labs/sasy-demo.git
   cd sasy-demo
   make setup
   ```

2. **Configure your keys**

   ```bash
   cp .env.example .env
   ```

   Edit `.env` and add your keys:

   ```bash
   OPENAI_API_KEY=sk-...
   SASY_API_KEY=demo-key-your-key-here
   ```

   Your SASY API key is provided separately.

</Steps>

## Run the Demo

The repo includes a pre-built Datalog policy
(`policy.dl`). Run the demo to see enforcement
in action:

<Tabs>
  <TabItem label="All scenarios">
    ```bash
    make demo
    ```
  </TabItem>
  <TabItem label="Interactive">
    ```bash
    make demo-step
    ```
    Press Enter between each stage.
  </TabItem>
  <TabItem label="Single scenario">
    ```bash
    make scenario-1
    ```
  </TabItem>
</Tabs>

This uploads `policy.dl` to the SASY policy engine
and runs 9 agent scenarios. You'll see:

```
[Customer] Hi, I need to cancel reservation RKLA42.

  → get_reservation_details(reservation_id=RKLA42)
    [SASY] Consulting policy engine...
    [SASY] ✓ AUTHORIZED: get_reservation_details

  → cancel_reservation(reservation_id=RKLA42)
    [SASY] Consulting policy engine...
    [SASY] ✗ DENIED: cancel_reservation
    Reason: No insurance, not a gold member
    → Consider adding travel insurance

[Agent] I'm sorry, your reservation cannot be
  cancelled because you don't have travel insurance.
```

## Translate a Policy

Write your policy in English (plus point the
translator at your agentic codebase) and get back a
Soufflé Datalog policy and companion C++ functors:

<Tabs>
  <TabItem label="SDK">
    ```python
    from sasy.policy import translate

    with open("policy_english.md") as f:
        policy = f.read()

    result = translate(
        policy,
        codebase_paths=["src/demo"],  # your agent
        codebase_root=".",
    )
    result.print_summary()
    result.save_all("output/", base_name="airline")
    ```
  </TabItem>
  <TabItem label="Makefile">
    ```bash
    make translate
    ```
  </TabItem>
</Tabs>

Translation takes ~5–15 minutes. Writes
`output/airline_policy.dl`,
`output/airline_functors.cpp`, and
`output/agent_summary.md`.

Upload and run the translated policy against the
scenarios:

```bash
make upload-translated    # ships output/airline_policy.dl
make demo-translated      # runs scenarios against it
```

See [Translate a Policy](/translate/) for the full
API and the [tau2 airline benchmark](/benchmarks/tau2-airline/)
for empirical results.

## Enforce a Policy

Here's how enforcement works in your own code.

### 1. Configure

Set `SASY_API_KEY` (and optionally `SASY_URL`) in
`.env` — the SDK reads them automatically on first
use. For programmatic overrides, call
`sasy.config.configure()`.

### 2. Upload

<Tabs>
  <TabItem label="SDK">
    ```python
    from sasy.policy import upload_policy_file

    resp = upload_policy_file("policy.dl")
    print("Uploaded!" if resp.accepted else resp.error_output)
    ```
  </TabItem>
  <TabItem label="Makefile">
    ```bash
    make upload              # uploads policy.dl (reference)
    make upload-translated   # uploads output/airline_policy.dl
    ```
  </TabItem>
</Tabs>

### 3. Check tool calls

Before executing any tool, check with the policy
engine:

```python
from sasy.reference_monitor import check_tool_call

auth = check_tool_call(
    fn_name="cancel_reservation",
    args='{"reservation_id": "RKLA42"}',
    # Pass the observability event IDs that led up to this
    # call (user message, prior tool results, etc.) when the
    # policy depends on message-graph context. For a bare
    # check with no prior context, pass None or [].
    input_node_ids=None,
)

if auth.authorized:
    # Execute the tool
    result = cancel_reservation(reservation_id="RKLA42")
else:
    # Policy denied — denial_trace has the human-readable
    # reasons + suggested remediation steps.
    reasons = [r.details for r in auth.denial_trace.reasons]
    print(f"Denied: {reasons}")
    print(f"Suggestions: {list(auth.denial_trace.suggested_fixes)}")
```

Every tool call your agent makes goes through
this check. The policy engine evaluates it against
the loaded Datalog and returns AUTHORIZED or
DENIED with a human-readable reason.

<Aside>
  The demo code in `src/demo/agent.py`
  shows the full integration — how tool calls are
  recorded in the dependency graph and checked
  against the policy.
</Aside>

### 4. Instrument (optional)

For automatic instrumentation of HTTP and agent
frameworks, one call is enough (auth comes from
`.env`):

```python
import sasy

sasy.instrument()  # patches HTTP, Langroid, etc.
```

This wraps HTTP requests and agent tool calls
so every action is automatically recorded and
checked — no manual `check_tool_call` needed.

See [How Enforcement Works](/demo/enforcement/)
for the full flow diagram.
