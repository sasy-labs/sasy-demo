import { Steps, Tabs, TabItem, Aside } from '@astrojs/starlight/components';

<Steps>

1. **Clone and install**

   ```bash
   git clone https://github.com/sasy-labs/sasy-demo.git
   cd sasy-demo
   make setup
   ```

2. **Configure your keys**

   ```bash
   cp .env.example .env
   ```

   Edit `.env` and add your keys:

   ```bash
   OPENAI_API_KEY=sk-...
   SASY_API_KEY=demo-key-your-key-here
   ```

   Your SASY API key is provided separately.

3. **Run the demo**

   The repo includes a pre-built Datalog policy
   (`policy.dl`) ready to use. Run it:

   <Tabs>
     <TabItem label="All scenarios">
       ```bash
       make demo
       ```
     </TabItem>
     <TabItem label="Interactive">
       ```bash
       make demo-step
       ```
       Press Enter between each stage to see what's
       happening step by step.
     </TabItem>
     <TabItem label="Single scenario">
       ```bash
       make scenario-1
       ```
     </TabItem>
   </Tabs>

   This uploads `policy.dl` to the SASY policy
   engine and runs 9 agent scenarios against it.

</Steps>

## Translate Your Own Policy

Once you've seen the demo, try translating a policy
from English yourself.

<Steps>

1. **Edit the English policy**

   Open `policy_english.md` and modify the rules
   (or write your own).

2. **Translate**

   ```bash
   make translate
   ```

   This sends the English to the SASY translation
   service and takes ~5-10 minutes. It produces:

   - `policy_compiled.dl` — the generated Datalog
   - `truth_table.tsv` — every test scenario

3. **Run the demo with your translated policy**

   ```bash
   make demo-compiled
   ```

   This uploads `policy_compiled.dl` (instead of
   the pre-built `policy.dl`) and runs all 9
   scenarios against your translated policy.

</Steps>

<Aside>
  See [Translate a Policy](/translate/) for details
  on the confidence report and verification artifacts.
</Aside>

## What You'll See

When you run a scenario, you'll see a narrated
walkthrough of the agent handling a customer
request:

```
[Customer] Hi, I need to cancel reservation RKLA42.

  → get_reservation_details(reservation_id=RKLA42)
    [SASY] Consulting policy engine...
    [SASY] ✓ AUTHORIZED: get_reservation_details

  → cancel_reservation(reservation_id=RKLA42)
    [SASY] Consulting policy engine...
    [SASY] ✗ DENIED: cancel_reservation
    Reason: No insurance, not a gold member
    → Consider adding travel insurance

[Agent] I'm sorry, your reservation cannot be
  cancelled because you don't have travel insurance.
```

Each step shows:
- **→** the tool call the agent is making
- **[SASY]** the policy engine checking the call
- **✓ AUTHORIZED** or **✗ DENIED** with the reason
- **[Agent]** how the agent responds to the customer
